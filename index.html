Benvenuti sulla pagina personale di Brischetto Salvatore Andrea. Di seguito sono riportati alcuni principi generali della cyber security:

  1. Confidenzialità: Assicurarsi che le informazioni siano accessibili solo a persone autorizzate. Utilizzare tecniche come la crittografia per proteggere i dati sensibili.
  2. Integrità: Garantire che le informazioni non siano alterate in modo non autorizzato o accidentale. Utilizzare firme digitali e controlli di integrità per rilevare modifiche non autorizzate ai dati.
  3. Disponibilità: Assicurarsi che le risorse informatiche siano sempre accessibili agli utenti autorizzati. Implementare misure per prevenire interruzioni, guasti o attacchi che potrebbero rendere inaccessibili i sistemi.
  4. Autenticazione: Verificare l'identità degli utenti e delle entità che accedono ai sistemi. Utilizzare password robuste, autenticazione a due fattori e altre tecniche per garantire che solo persone autorizzate possano accedere ai dati e ai servizi.
  5. Autorizzazione: Definire i livelli di accesso appropriati per gli utenti e le entità in base alle loro responsabilità e privilegi. Limitare l'accesso solo alle risorse necessarie per svolgere il proprio lavoro.
  6. Responsabilità: Ogni utente deve essere responsabile delle proprie azioni digitali. Mantenere le password segrete, evitare di condividere informazioni sensibili e utilizzare software legalmente acquisito.

La cyber security è di fondamentale importanza per proteggere le informazioni personali, aziendali e sensibili da minacce informatiche sempre più sofisticate. Utilizzando i principi sopra citati e adottando buone pratiche di sicurezza informatica, possiamo contribuire a creare un ambiente online più sicuro per tutti.

Per ulteriori informazioni sulla cyber security e sulle migliori pratiche, si consiglia di consultare siti web e risorse affidabili come organizzazioni specializzate, enti governativi e community di esperti di sicurezza informatica.

Di seguito sono riportati alcuni principali programmi e progetti che ho realizzato:

  • AppFlask

    • Questo codice Flask definisce un'applicazione web con una sola pagina. Quando si accede alla radice dell'applicazione, viene visualizzato il template 'index.html'. Se viene inviato un modulo tramite una richiesta POST, viene estratto il valore del campo username e viene restituito il template 'welcome.html' con il nome utente inserito. Si consiglia di creare i file HTML 'index.html' e 'welcome.html' nella stessa directory del file Python per fornire il contenuto desiderato per le pagine.

  • Cookie

    • Questo codice Flask definisce un'applicazione web con funzionalità di login/logout utilizzando le sessioni. Quando si accede alla radice dell'applicazione, viene controllato se l'utente è già autenticato e viene visualizzato un messaggio di benvenuto personalizzato se lo è. Altrimenti, viene visualizzato un modulo di login. Quando viene effettuato il login, l'username viene salvato nella sessione insieme all'ultimo accesso. È possibile effettuare il logout per rimuovere l'username e l'ultimo accesso dalla sessione. Si consiglia di creare il file HTML 'login.html' nella stessa directory del file Python per fornire il modulo di login personalizzato.

    Assicurati di mantenere la chiave segreta (app.secret_key) protetta e di non divulgarla pubblicamente, poiché è utilizzata per firmare la sessione e proteggere le informazioni.
  • ServiceScan

    • Questo codice è uno script Python per la scansione dei servizi su un determinato indirizzo IP.Si consiglia di personalizzare l'indirizzo IP del target (target_ip),in questo esempio io ho usato l'IP di Google, la porta di partenza (start_port) e la porta di fine (end_port) in base alle tue esigenze di scansione dei servizi.

  • TestApp

    • In sintesi, questo codice Flask definisce un'applicazione web che mostra un form di accesso. Quando si invia il form con l'username e la password, vengono visualizzati i valori ricevuti. Altrimenti, viene visualizzato il form per l'inserimento dei dati di accesso. Si consiglia di personalizzare ulteriormente l'applicazione aggiungendo la logica di autenticazione o l'invio dei dati di accesso a un backend per la verifica delle credenziali."
